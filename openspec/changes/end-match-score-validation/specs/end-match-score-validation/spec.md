## ADDED Requirements

### Requirement: 结束对局前校验筹码差额
管理员点击"结束对局"时，系统 SHALL 计算所有已填写结算筹码的玩家的差额值：`diff = sum(finalChips) - sum(bonus)`。若 `|diff| >= 0.01`，系统 MUST 阻止进入原有流程，并展示差额错误弹窗。

#### Scenario: 差额不为零时阻断结束
- **WHEN** 管理员点击"结束对局"，且 `sum(finalChips) - sum(bonus) !== 0`（绝对值 ≥ 0.01）
- **THEN** 系统不弹出原有的未填写确认弹窗，而是弹出差额错误弹窗，显示差额数值（如 `+50` 或 `-30`）

#### Scenario: 差额为零时正常流程
- **WHEN** 管理员点击"结束对局"，且 `sum(finalChips) - sum(bonus) === 0`（绝对值 < 0.01）
- **THEN** 系统继续原有的未填写人数确认流程，不显示差额错误弹窗

#### Scenario: 全员未填写时跳过差额校验
- **WHEN** 管理员点击"结束对局"，且所有玩家的 `finalChips` 均为 null 或 undefined
- **THEN** 差额视为 0，系统继续原有流程

### Requirement: 差额错误弹窗展示内容
差额错误弹窗 SHALL 包含：错误标题、当前差额数值（带正负符号）、关闭按钮。弹窗 MUST NOT 提供"仍然结束"选项，只允许用户返回修改数据。

#### Scenario: 弹窗显示正差额
- **WHEN** `diff > 0`（多出筹码）
- **THEN** 弹窗显示类似"差额 +50，请检查各玩家结算积分"的提示

#### Scenario: 弹窗显示负差额
- **WHEN** `diff < 0`（少了筹码）
- **THEN** 弹窗显示类似"差额 -30，请检查各玩家结算积分"的提示

#### Scenario: 用户关闭差额弹窗
- **WHEN** 用户点击差额错误弹窗的关闭/确认按钮
- **THEN** 弹窗关闭，页面回到对局详情，用户可继续编辑积分
